# Implement the configuration loading layer in Go (`pkg/config`) to find, read, and parse `~/.agent/config.yml`, extracting named adapter configurations and supporting environment variable overrides for custom configuration paths, with robust error handling for missing or malformed configuration files.

This task involves implementing the core configuration loading module under the `pkg/config` package in Go. The config loader must resolve the location of the user's home directory to find and read the `~/.agent/config.yml` configuration file. It will parse the YAML file, which maps user-defined, named adapters (like 'primary', 'fast', or 'claude') to their target execution specification strings.

The implementation must handle missing configuration files gracefully, returning clean instructions on how to initialize the configuration. Additionally, it should support config path overrides via environment variables (such as `AGENT_CONFIG_PATH`) to allow dynamic configuration path resolution. The loader must be robust against file access issues and parse YAML structures cleanly, ensuring key/value entries are correctly extracted and ready for the downstream adapter parser.
