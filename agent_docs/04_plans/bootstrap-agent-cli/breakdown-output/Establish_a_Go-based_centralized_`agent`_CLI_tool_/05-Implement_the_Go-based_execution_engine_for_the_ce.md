# Implement the Go-based execution engine for the centralized agent router by defining the Runner interface and its routing layer, and fully implement the Opencode driver runner to execute the under-the-hood opencode CLI subprocess, parsing and aggregating its streaming JSON-formatted output into a clean response.

This task establishes the core execution layer of the agent CLI tool. It begins by defining the common Runner interface, which dictates the contract for all backend driver runners. Each runner must implement a Run method that accepts an execution context, the target model name, and the input prompt, and returns the aggregated text response or an error. A unified runner registry or routing mechanism will be created to resolve the correct Runner implementation dynamically based on the parsed adapter specification (for example, mapping the 'opencode' driver name to the OpencodeRunner).

The primary implementation target within this task is the OpencodeRunner. This driver is responsible for launching the external opencode executable as an OS subprocess with the appropriate arguments, specifically 'opencode run <prompt> --model <model> --format json'. Because the output is streamed as a sequence of JSON objects, the runner must monitor the subprocess's standard output, parse each stream fragment (representing tokens) in real-time, and aggregate the raw content.

Additionally, the runner must handle subprocess exit codes and capture stderr to report execution errors gracefully. The completed runner and routing mechanism will plug directly into the CLI's main entry point, providing a complete and testable integration slice for the Opencode CLI driver.
