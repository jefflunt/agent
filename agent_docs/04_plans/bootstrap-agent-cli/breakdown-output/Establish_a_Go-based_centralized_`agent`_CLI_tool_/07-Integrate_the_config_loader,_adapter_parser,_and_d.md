# Integrate the config loader, adapter parser, and driver runner modules inside `main.go` to deliver a fully functional, end-to-end command-line pipe router. The application must process positional arguments for target adapter profiles, consume input prompts from STDIN, execute the matching driver runner subprocess, stream clean responses to STDOUT, and compile cleanly into the centralized `agent` binary.

This task focuses on wiring together the independent modules—specifically the profile configuration loader, the adapter specification parser, and the runner router—inside the application's CLI entrypoint (main.go). The entrypoint must parse positional arguments to determine the active configuration profile, read standard input (STDIN) to buffer the prompt payload, fetch the associated adapter mapping, route the request to the corresponding runner subprocess (such as opencode or copilot), and pipe the sanitized results directly to standard output (STDOUT).

Additionally, this task involves verifying that the entire Go codebase compiles end-to-end into a single 'agent' binary. It must support clean compilation, utilize correct imports across package boundaries, handle execution errors gracefully by outputting descriptive errors to standard error (STDERR), and return appropriate Unix exit codes.
