# Implement the new configuration schema in `pkg/config/config.go` by introducing the `AdapterConfig` struct with a custom `yaml.Unmarshaler` implementation. This custom logic must support both simple scalar string values (for backward compatibility) and nested structures containing a `target` string and a list of custom CLI `flags`. Update the main `Config` type to map adapter names to the new `AdapterConfig` representation.

This task involves introducing a new structured configuration schema in `pkg/config/config.go` to support customizable per-LLM driver flags. Currently, the `Config` struct parses the `adapters` field into a simple `map[string]string` format. We will replace this with a `map[string]AdapterConfig` representation.

To ensure backward compatibility, the `AdapterConfig` struct must support custom YAML unmarshaling by implementing the `yaml.Unmarshaler` interface. This custom logic will inspect the incoming YAML node. If the node is a scalar string, it should decode the value into the `Target` field. If the node is a mapping object, it should decode both the destination target and the custom flags slice. Finally, the main configuration parsing logic (`ParseConfig`) must be updated to load this new representation.
